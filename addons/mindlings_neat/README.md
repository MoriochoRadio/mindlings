# mindlings_neat

Mindlings 전용 경량 NEAT 신경망 라이브러리.

## 왜 직접 구현했나 (M2 결정 기록)

TECH_STACK 2장의 후보들을 평가한 결과:

| 후보 | 성격 | 우리와의 적합성 |
|---|---|---|
| Godot-AI-Kit | GDScript 네이티브 AI 모음(NEAT/DQN 등) | NEAT가 배치형 GA 루프 중심. 좋은 참고자료 |
| NEAT_GDScript (asset #987) | 두 클래스 상속 → NEAT 자동 구성 | 전역 세대 GA가 우리 모델과 충돌 |
| neat4godot | Godot용 NEAT 구현 | 동일하게 배치형 세대 진화 전제 |

**핵심 불일치**: 위 라이브러리는 *고정 인구 → 적합도 평가 → 다음 세대*의 배치형
유전 알고리즘을 가정한다. 반면 Mindlings(GAME_DESIGN 4장)는 개체가 살아가며
에너지가 차면 그 자리에서 번식하는 **유기적·연속 진화**로, 전역 세대 장벽이 없다.

TECH_STACK 2장은 이 경우를 예견해 *"핵심 NEAT 로직은 직접 이해·수정할 수 있도록
얇게 감싼다. 작은 MLP 수준이라 직접 구현도 현실적"* 이라고 명시한다. 그래서:

- **채택**: NEAT 양식(노드/연결 유전자)을 따르는 자체 경량 구현.
- **이점**: 외부 의존·라이선스 리스크 0(무료 오픈소스 원칙), 디버깅 용이,
  우리 생애주기(개체별 두뇌·연속 번식)에 정확히 맞음.
- **확장성**: 일반적 위상 정렬 forward 평가를 갖춰, M3에서 돌연변이로 은닉
  노드·연결을 키워도 그대로 동작한다.

## 구성

- `neural_network.gd` (`class_name MindNet`) — 노드/연결 유전자, forward pass(`set_inputs`
  → `propagate` → `get_outputs`), `clone()`(M3 유전 대비).

게임 쪽 배선(센서 8 + 출력 3, 초기 가중치)은 `scripts/brain_builder.gd`에 있다.

## 로드맵

- **M2 (현재)**: 최소 토폴로지(은닉 없음, 입력→출력 직결). 진화 전이라
  의도적 행동이 보이도록 약한 사전 편향 + 무작위 가중치.
- **M3**: 번식 시 부모 망 복제 후 돌연변이(가중치 변이 + 노드/연결 추가) →
  구조가 세대마다 자라난다.
