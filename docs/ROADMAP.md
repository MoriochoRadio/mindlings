# Mindlings — 개발 로드맵 & Claude Code 작업 명세

> 사용법: 각 마일스톤은 순서대로 진행. 작업을 시킬 땐 Claude Code에게
> **"ROADMAP.md의 M2를 구현해줘"** 처럼 마일스톤 번호로 지시하면 된다.
> Code는 이 문서 + GAME_DESIGN.md + CLAUDE.md를 함께 읽고 작업한다.
> 큰 설계 결정이나 막힘이 생기면 Cowork(설계·리뷰)로 돌아온다.
> 최종 수정: 2026-06-13

---

## 진행 원칙
- **"동작하는 가장 작은 것"을 매 단계 만든다.** 각 M는 실행 가능한 결과물로 끝난다.
- 매 마일스톤 종료 시: 실행 확인 → 의미 있는 커밋 → 필요 시 Cowork 리뷰.
- 범위가 흔들리면 GAME_DESIGN.md 5장(MVP 한계선)으로 돌아온다.

---

## M0 — 프로젝트 부트스트랩
**목표**: 빈 Godot 프로젝트가 GitHub에 올라가고 실행된다.

Claude Code 작업 명세:
- Godot 4.x 프로젝트 생성(2D). 프로젝트명/설정 기본값.
- 디렉토리 구조 생성: `scenes/`, `scripts/`, `assets/`, `docs/`(기존), `addons/`.
- `.gitignore`(Godot용: `.godot/`, 빌드 산출물 등) 추가.
- 빈 메인 씬(`scenes/Main.tscn`) + 빈 `Main.gd` 생성, 실행 시 빈 화면 뜨는지 확인.
- README.md 초안(프로젝트 한 줄 소개 + docs 링크).
- 첫 커밋 & 푸시.

**완료 기준**: `git clone` 후 Godot에서 열어 F5로 빈 씬이 실행됨.

## M1 — 세계와 개체(뇌 없음)
**목표**: 화면에 움직이는 생명체와 먹이가 있다. (아직 신경망 없음, 랜덤/규칙 이동)

Claude Code 작업 명세:
- `World` 노드: 2D 경계 영역, 배경.
- `Creature` 씬/스크립트: 도형으로 표현, 에너지 스탯, 시간당 에너지 감소,
  0이 되면 사망(제거). 일단 랜덤 워크로 이동.
- `Food` 씬: 맵에 주기적으로 스폰. 개체가 닿으면 먹고 에너지 획득.
- 간단한 스폰 매니저(개체 N마리, 먹이 주기적 생성).
- 시간 배속 토글(일시정지/1x/2x/5x)과 개체 수 표시 라벨.

**완료 기준**: 개체들이 돌아다니며 먹이를 먹고, 굶으면 죽는다.

---

## M2 — NEAT 신경망 이식 & 개체 두뇌 연결
**목표**: 개체가 신경망으로 움직인다. (아직 진화는 단순)

Claude Code 작업 명세:
- NEAT 라이브러리 선정·`addons/`에 통합 (TECH_STACK.md 2장 후보 검토).
  먼저 **Godot-AI-Kit** 또는 **NEAT_GDScript**를 평가해 하나 채택. 이유를 커밋/문서에 기록.
- 개체 센서(입력) 구현: 가장 가까운 먹이의 방향·거리, 자기 에너지, 가까운 동족 밀도 등
  8개 내외 입력.
- 신경망 출력 → 개체 행동 매핑: 이동 방향/속도, 먹기 시도.
- 개체마다 개별 신경망 인스턴스를 갖고, 매 틱 forward pass로 행동 결정.

**완료 기준**: 개체가 (랜덤이 아니라) 신경망 출력에 따라 먹이로 향하는 등 의도적으로 보임.

---

## M3 — 번식 & 진화 루프
**목표**: 세대가 돌고, 종이 점점 먹이를 잘 찾도록 진화한다.

Claude Code 작업 명세:
- 번식: 에너지 임계치 넘으면 자식 생성, 에너지 일부 소모.
- 유전: 부모 신경망(가중치+구조)을 복제 후 돌연변이(NEAT 변이 연산) 적용.
- 인구 동역학 안정화: 먹이량·에너지·돌연변이율 파라미터를 `export`로 노출(튜닝용).
- 통계 수집: 세대수, 평균 수명, 인구, 평균 신경망 크기(노드/연결 수).

**완료 기준**: 수십~수백 세대 돌렸을 때 평균 적합도(생존·먹이 효율)가 상승하는 경향 관찰.

---

## M4 — 신의 간섭 (Core Loop 완성)
**목표**: 플레이어가 세계에 개입할 수 있다. (게임의 핵심 재미)

Claude Code 작업 명세 (GAME_DESIGN 3장 기준 4~6종):
- 먹이 뿌리기 / 줄이기 (마우스로 영역 지정).
- 포식자 풀기 (개체를 사냥하는 단순 AI 엔티티).
- 장애물/지형 추가·제거.
- 개체 집기(드래그로 이동)·제거.
- 재해 1~2종(예: 가뭄 = 먹이 스폰 일시 급감).
- 간섭 UI 툴바(아이콘 + 단축키).

**완료 기준**: 플레이어 간섭에 생태계가 단기 반응하고, 장기적으로 진화 방향이 달라진다.

---

## M5 — 관찰 도구 & 신경망 시각화 (셀링 포인트)
**목표**: 개체를 클릭하면 그 뇌가 보인다. 데이터로 드라마를 만든다.

Claude Code 작업 명세:
- 개체 클릭 → 정보 패널(나이/에너지/세대/계보) + **신경망 그래프 실시간 시각화**
  (노드·연결, 발화 강도 색/굵기).
- 인구·세대·평균 신경망 크기 등 **시계열 그래프** 패널.
- 데이터 오버레이 토글: 센서 시야, 먹이 히트맵 등.
- "이정표 알림"(첫 은닉 노드 출현, 평균 수명 2배 등) 토스트 메시지.

**완료 기준**: 클릭 한 번으로 한 개체의 신경망이 실시간 발화하는 화면을 볼 수 있다(트레일러용).

---

## M6 — 세이브/로드 & 안정화
**목표**: 월드를 저장/복구하고, 성능·버그를 잡는다.

Claude Code 작업 명세:
- 월드 상태 직렬화(개체 신경망 포함) 저장/로드.
- 성능 프로파일링: 개체 수 상한, 고정 timestep 검토, 병목 시 최적화(C#/멀티스레드 후보).
- 버그·예외 처리, 기본 설정 메뉴(배속·파라미터·해상도).

**완료 기준**: 저장 후 종료→재시작→로드 시 동일 생태가 복구된다. 목표 개체 수에서 안정 fps.

---

## M7 — 플레이어 동기 & 콘텐츠
**목표**: "그래서 뭘 하지?"를 해결한다. (GAME_DESIGN 7장)

Claude Code 작업 명세:
- 도전 과제 시스템(예: N세대 생존, 특정 행동 유도) 3~5개.
- 타임랩스/진화사 앨범 기록(공유용 스크린샷).
- 온보딩 튜토리얼(간단 안내) + 기본 UX 다듬기.

**완료 기준**: 신규 플레이어가 설명 없이도 핵심 루프를 30분 즐길 수 있다.

---

## M8 — Steam 통합 & 출시 준비
**목표**: 스팀에 올릴 수 있는 빌드.

Claude Code 작업 명세:
- **GodotSteam** 통합(실적·클라우드 세이브·통계 일부).
- Windows 익스포트 빌드 파이프라인 정리, 빌드 산출물 점검.
- (병행, 사람이 할 일) Steamworks 가입 → $100 결제 → 스토어 페이지(스크린샷/트레일러/
  설명/태그) 작성 → 빌드 업로드 → 검수 → 출시일.

**완료 기준**: 스팀 검수 통과 가능한 빌드 + 완성된 스토어 페이지.

> 비용·절차 상세는 TECH_STACK.md 4장 참고. 출시 임박 시 Steam 정책 재확인.

## 작업 분담 요약
- **Cowork(나)**: 설계 결정, 문서 갱신, 코드/구조 리뷰, 막힘 해소, 범위 관리.
- **Claude Code**: 위 명세대로 실제 구현·실행·테스트·커밋.
- **사람(개발자)**: Code에 마일스톤 지시, Godot 에디터 확인, Steam 계정/결제/스토어 페이지.
