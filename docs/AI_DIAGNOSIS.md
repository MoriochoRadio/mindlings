# AI 기여도 진단 리포트 (2026-07-02)

> 목적: "이 게임의 개체를 실제로 움직이는 것은 무엇인가 — 신경망 학습·진화인가,
> 아니면 사람이 손으로 심은 규칙인가?"를 감이 아니라 **데이터로** 확인한다.
> 배경: 핵심 셀링("The Sims인데 캐릭터가 진짜로 학습하고 생각한다")이 실제 구현으로
> 뒷받침되는지에 대한 근본적 의문에서 출발.

## 방법

행동을 결정하는 세 축을 독립적으로 켜고 끄며 헤드리스 배치 시뮬레이션:
- **본능**(`instinct_strength`): 창시자 신경망에 심은 사전 가중치(먹이·회피·물 등).
- **안전망**(`survival_scaffolding`): 매 틱 하드코딩 생존 보정(갈증 반사 드라이브,
  생활권 복귀, 욕구 재확립 복귀, 만족-상태 친구 곁 모임).
- **학습**(`learning_enabled`, `learning_rate`, `weight_homeostasis`): 생애 내 보상조절 가소성.

계측: 시뮬 120초(배속 8x) × 시드 3개. 인구·세대·평균수명·채집량·멸종시점.
재현: `MINDLINGS_DIAG=1 MINDLINGS_INSTINCT=.. MINDLINGS_SCAFFOLD=.. MINDLINGS_LEARN=..`
환경변수로 `godot --headless` 실행(하네스는 `World.gd`의 `_apply_diag_env`/`_diag_tick`).

## 결론 (데이터 확증)

1. **생존을 결정하는 유일한 요인은 창시자 본능 가중치.** 있으면 항상 안정, 없으면
   항상 ~50초에 전멸. 안전망·학습·포식 유무가 이 결과를 바꾸지 못한다.
2. **생애 학습의 실질 기여 ≈ 0.** 학습 ON/OFF, 20배로 풀어줘도 결과 불변.
   원인은 **부트스트랩 실패** — 백지 개체는 먹으러 갈 줄 몰라 보상 자체가 발생하지
   않는다(먹어야 학습하는데 먹는 법을 학습으로 못 만든다). 개발 중 학습 폭주→전멸을
   가중치 항상성으로 눌렀고, 그 결과 "불안정해서 껐더니 무의미해진" 악순환.
3. **매 틱 안전망은 불필요하며 역효과.** 끄면 채집·수명이 오른다. 본능이 안정성을
   이미 100% 책임지므로 안전망은 행동을 제약만 했다. 포식 압력에서도 없이 안정적.

## 방향 결정: C. 하이브리드

- 셀링을 정직하게 조정: "진짜 학습 AI"를 과장하지 않고, 실제 강점(**본능+진화로
  살아가는 작은 사람들 + 마음이 다 읽히는 가독성 + 능동적 신 간섭 + 관계 드라마**)으로.
- 학습을 '생존 엔진'이 아니라 **'읽히는 습관·선호'**로 재정의(생존을 좌우하지 않되,
  눈에 보이는 성격 형성으로 — 부트스트랩 문제를 우회).
- 데이터가 지지하는 즉시 개선: 불필요한 안전망 제거 → 코드 단순화 + 창발 여지 회복.

## 한계(공정성)

- 포식자 없는/2마리 기본 세계 기준. 훨씬 복잡한 환경에선 학습 기여가 달라질 수 있음.
- 120초/8배속으론 2–7세대까지만 진행 → 장기 진화 효과는 이 진단의 범위 밖.
- "인구 안정"은 계측했으나 "표류 사망이 눈에 거슬리는가" 같은 UX는 GUI 관찰 필요.
